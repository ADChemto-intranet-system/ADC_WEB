## 버전 관리 규칙

### 버전 번호 체계
- **형식**: v[주버전].[부버전].[수정버전]
- **예시**: v1.0.0, v1.1.0, v1.1.1

### 버전 업데이트 기준
- **주버전 (Major)**: 대규모 기능 추가, 아키텍처 변경
- **부버전 (Minor)**: 새로운 기능 추가, 기존 기능 개선
- **수정버전 (Patch)**: 버그 수정, 소규모 개선
